# AI Public API Stability Checklist

The refactor must keep the legacy `engine.ai.AI` surface 100% backward compatible. The
following constructors, methods, getters (generated by Lombok), and constants may not
change signature, visibility, or semantics.

## Constructors

- `AI(Engine mainEngine)`

## Static constants

- `int MIN_HASH_SIZE_MB`
- `int MAX_HASH_SIZE_MB`
- `double EXIT_FLAG`
- `long INFINITE_TIME_LIMIT`

## Lombok-generated getters

- `Engine getMainEngine()`
- `int getSearchThreads()`
- `int getLazySmpThreads()`
- `int getHashSizeMb()`
- `int getTranspositionTableCapacity()`
- `int getCaptureTranspositionTableCapacity()`
- `int getMaxDepth()`
- `long getTimeLimit()`
- `long getNodesVisited()`
- `long getNullMoveCount()`
- `SearchDiagnostics getLastDiagnostics()`

## Explicit accessors / mutators

- `List<MoveAndScore> getCalculatedLine()`
- `void setHashSizeMb(int hashSizeMb)`
- `void setMaxDepth(int depth)`
- `void setTimeLimit(long timeLimitMillis)`
- `void setSearchThreads(int requestedThreads)`

## Search control & status

- `Integer getCurrentBestMoveInt()`
- `void reset()`
- `long getSearchElapsedMillis()`
- `void stopCalculation()`
- `void requestStop()`
- `void startAutoPlay(boolean aiIsWhite, boolean aiIsBlack)`
- `void performMove()`
- `double evaluateBoard(Engine simulatorEngine, boolean isWhitesTurn, long deadline)`
- `void updateBoardStateHash()`

Any additional helper surfaces introduced by the refactor must be kept internal to the
new search packages so that the stable API remains unchanged.
